# 知桥AI开发计划

## 概述

知桥 AI 面向 K12 教学场景，以教材知识点为纽带，分阶段交付 Web 产品。本计划与 [项目愿景](./project-vision.md)、[V1.0 路线图](./v1-roadmap.md)、[产品架构](./product-architecture.md) 对齐：先搭平台与双端入口，再接入 AI 能力，最后闭环学情与知识库。

---

## 第一阶段：平台与双端基础

**目标**：可登录、可区分角色的 Web 应用，教师端与学生端各有清晰首页与导航骨架。

### 范围

- Web 版本（响应式布局，优先桌面端备课与平板/手机端学习）
- 教师端首页：快捷入口（备课、学情占位）、最近使用、公告区
- 学生端首页：学习入口（答疑、复习占位）、今日任务、进度概览
- 登录系统：注册/登录、角色（教师/学生）、会话保持与退出

### 技术任务

| 任务 | 说明 |
|------|------|
| 项目初始化 | Next.js App Router + TypeScript，目录划分 `app/`、`components/`、`lib/` |
| UI 基础 | 布局、导航、主题色与组件库选型（可与 shadcn/ui 等配合） |
| Supabase 接入 | 项目创建、环境变量、Auth（邮箱或手机号按实际需求） |
| 用户与角色 | `profiles` 表扩展角色字段；路由/中间件按角色跳转教师端或学生端 |
| 双端路由 | `/teacher`、`/student` 及子路由占位；未登录重定向登录页 |

### 交付物

- 可部署的 Web 应用（开发/预发环境）
- 教师、学生各至少 1 个可演示账号
- 双端首页静态或 Mock 数据展示，无 AI 依赖即可验收

### 验收标准

- 教师/学生登录后进入对应首页，互不越权
- 移动端基础可用（导航不溢出、主要按钮可点）
- 环境配置文档齐全（`.env.example`）

### 预估周期

2–3 周（视 UI 精细度与 Auth 方案而定）

---

## 第二阶段：AI 核心能力

**目标**：基于知识库与 Prompt 模板，上线教师备课与学生答疑三条主链路。

### 范围

- AI 教案生成：对接 [教案 Prompt](../prompts/lesson-generator.md)，输入学科/年级/版本/单元/知识点，输出结构化教案
- AI 知识点讲解：面向学生的分步讲解、例题与小结
- AI 答疑：学生提问，结合当前学科与知识点上下文回答

### 技术任务

| 任务 | 说明 |
|------|------|
| DeepSeek API | 服务端 Route Handler 调用，密钥仅存服务端；流式输出可选；使用 OpenAI 兼容客户端 |
| Prompt 管理 | `prompts/` 目录版本化；参数注入与输出 JSON/Markdown 解析 |
| 知识库最小集 | Supabase 表：教材、单元、知识点（可先手工录入种子数据） |
| 生成记录 | 教案、讲解、答疑会话落库，供教师回看与学生历史 |
| 教师端页面 | 教案生成表单、结果展示、复制/导出 |
| 学生端页面 | 答疑对话 UI、知识点讲解入口 |

### 数据与流程

```
教材/知识点（Supabase）→ 组装 Prompt → DeepSeek → 结果存储 → 教师端/学生端展示
```

### 交付物

- 三条 AI 功能可端到端演示
- 至少 1 套学科（如初中数学）种子知识点数据
- API 错误与超时基础处理（用户可读提示）

### 验收标准

- 教案输出包含：目标、重点、难点、流程、互动、巩固（与 Prompt 一致）
- 答疑能引用或约束在指定知识点范围内（配置项可控）
- 单次生成有记录可查（用户 ID、时间、输入摘要）

### 依赖

- 第一阶段登录与角色体系完成
- DeepSeek API 配额与模型选型确定（默认 `deepseek-v4-flash`，高阶 `deepseek-v4-pro`）

### 预估周期

3–4 周

---

## 第三阶段：数据闭环与运营能力

**目标**：沉淀学习数据，支持学情洞察、错题复习与知识库可维护。

### 范围

- 学情分析：班级/学生维度掌握度、薄弱知识点、练习完成情况（教师端看板）
- 错题系统：自动或手动收录错题，关联知识点，支持再练
- 知识库管理：教材、知识点、题目、教学案例的增删改查（教师或管理员）

### 技术任务

| 任务 | 说明 |
|------|------|
| 学习事件埋点 | 答题、答疑、复习、教案使用等行为写入 `learning_events` |
| 统计聚合 | 按知识点汇总正确率、频次；定时或按需计算 |
| 学情看板 | 教师端图表与列表（可先表格 + 简单图表） |
| 错题表结构 | 题目快照、知识点 ID、错误次数、掌握状态 |
| 知识库后台 | CRUD 界面或简化管理页；权限：教师只读/编辑范围按产品设计 |
| 数据回流 | 高频错题、高频答疑问题反哺知识点标签或推荐（规则优先，AI 增强可后续） |

### 交付物

- 教师学情页（班级列表 → 学生详情 → 知识点掌握）
- 学生错题本与「再练一次」
- 知识库管理最小可用（教材树 + 知识点 + 题目录入）

### 验收标准

- 学情数据与真实学习行为一致（可追溯事件源）
- 错题与知识点、题目可关联查询
- 知识库变更后，AI 生成可读到最新知识点（缓存策略需说明）

### 依赖

- 第二阶段已产生答疑、练习等行为数据
- 题目数据结构在第二阶段或本阶段初定稿

### 预估周期

4–5 周

## 技术路线

| 层级 | 选型 | 用途 |
|------|------|------|
| 前端框架 | Next.js | SSR/路由、API Route、统一部署 |
| 语言 | TypeScript | 类型安全、前后端共享类型 |
| 后端与数据 | Supabase | Auth、PostgreSQL、Realtime（按需）、Storage（课件/附件后续） |
| AI | DeepSeek API | 教案、讲解、答疑生成 |
| 部署 | Vercel 或同类 | 与 Next.js 生态契合；环境变量管理 |

### 建议目录结构（随阶段演进）

```
app/
  (auth)/          # 登录注册
  teacher/         # 教师端
  student/         # 学生端
  api/             # AI 与业务 API
components/
lib/               # Supabase client、DeepSeek（OpenAI 兼容）封装
prompts/           # Prompt 模板
docs/              # 产品与开发文档
```

### AI 服务（DeepSeek）

| 配置项 | 值 |
|--------|-----|
| Provider | DeepSeek |
| Base URL | `https://api.deepseek.com` |
| 默认模型 | `deepseek-v4-flash`（答疑、轻量生成、高频调用） |
| 高阶模型 | `deepseek-v4-pro`（教案、长文本、复杂讲解） |

> **说明**：DeepSeek API 与 [OpenAI SDK](https://github.com/openai/openai-node) 兼容，项目可通过 OpenAI 兼容客户端配置接入（设置 `baseURL` 与 `apiKey`），无需单独引入另一套 SDK。

建议环境变量（`.env.example`）：

```
DEEPSEEK_API_KEY=
DEEPSEEK_BASE_URL=https://api.deepseek.com
DEEPSEEK_MODEL_DEFAULT=deepseek-v4-flash
DEEPSEEK_MODEL_ADVANCED=deepseek-v4-pro
```

### 安全与合规要点

- API Key、Supabase Service Role 仅服务端使用
- 学生数据按班级/学校隔离（RLS 策略在 Supabase 配置）
- AI 输出需标注「辅助生成，请教师审核后使用」
- 敏感信息不入 Prompt（姓名、身份证等）

## 里程碑总览

| 阶段 | 里程碑 | 关键产出 |
|------|--------|----------|
| 一 | M1 平台可用 | 登录 + 双端首页 |
| 二 | M2 AI 上线 | 教案 + 讲解 + 答疑 |
| 三 | M3 数据闭环 | 学情 + 错题 + 知识库管理 |

**V1.0 整体目标**（与路线图一致）：在 Web 上完成教师备课与学生学习的核心闭环；小程序与 App 不在本计划内。

---

## 风险与应对

| 风险 | 应对 |
|------|------|
| AI 成本与延迟 | 默认轻量模型；长教案可分段生成；限流与每日配额 |
| 知识点数据不足 | 先单学科试点；人工校对种子库再扩科 |
| 学情指标口径不一 | 第三阶段前冻结事件字典与统计口径文档 |
| 范围蔓延 | 严格按三阶段验收；课件生成、每日复习等列入 V1.1+ |

---
